Performance

Indexing Strategy

6 non-clustered indexes optimize our most frequent query patterns

  • 📍 IX_Devices_Location — Fast device lookup by room
  • 🔌 IX_Devices_Status — Quick filter for online/offline devices
  • 📊 IX_EnergyUsage_Device_Timestamp — Optimized time-series queries
  • 🚨 IX_Alerts_ResolutionStatus — Fast pending alert retrieval
  • ⚠️ IX_Alerts_Severity — Priority-based alert filtering
  • ⚙️ IX_AutomationRules_DeviceID — Quick rule lookup per device

💡 Why Indexes Matter

Without indexes, SQL Server performs full table scans. Our indexes enable seek operations that are orders of magnitude faster — critical for a system processing real-time sensor data.

⚡ Composite index on (DeviceID, Timestamp) enables efficient range queries for energy reports

Data Protection

Backup & Recovery

Full Recovery Model ensures zero data loss for continuous IoT telemetry

🗄️ Full Backup — Daily at 2:00 AM

Complete database snapshot as the recovery baseline

📦 Differential — Every 4-6 hours

Captures only changes since last full backup

📋 Transaction Log — Every 15-30 min

Enables point-in-time recovery to the exact minute

🔥 Disaster Recovery Scenario

Server failure at 4:20 PM:

  • Step 1: Restore Full Backup (2:00 AM) with NORECOVERY
  • Step 2: Restore Differential (2:00 PM) with NORECOVERY
  • Step 3: Restore Transaction Logs (2:15–4:15 PM) with RECOVERY

✅ Database restored to 4:15 PM — only 5 minutes of data loss
